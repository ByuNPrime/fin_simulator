# 金融精英模拟器

一款模拟金融职场生涯的数值策略游戏，从初级分析师起步，在高压金融环境中求生存、谋发展。

## 快速开始

### 方法一：直接打开（推荐）
1. 双击 `index.html` 文件
2. 游戏将在浏览器中打开，即可开始游玩

### 方法二：使用本地服务器
如果你安装了Python，可以在项目目录运行：

```bash
# Python 3
python -m http.server 8000

# Python 2
python -m SimpleHTTPServer 8000
```

然后在浏览器访问：`http://localhost:8000`

## 游戏操作指南

### 基础玩法
1. **行动点系统**：每个季度有4个行动点（AP）
2. **选择行动**：点击右侧行动面板中的按钮消耗AP执行行动
3. **结束季度**：用完所有行动点后，点击"结束本季度"按钮
4. **应对事件**：随机事件会弹出，选择不同选项会有不同后果
5. **目标**：避免失败条件，达成胜利条件

### 核心数值说明
| 数值 | 说明 | 影响 |
|------|------|------|
| 精力储备 | 你的体力值 | 归零则游戏失败 |
| 职业声誉 | 行业内的名声 | 影响项目和晋升 |
| 高层关系 | 与领导的关系 | 影响晋升成功率 |
| 业内人脉 | 行业人际关系 | 影响资源和机会 |
| 专业能力 | 你的专业水平 | 影响项目成功率 |
| 业绩表现 | 季度业绩成果 | 年度清零，影响晋升和奖金 |
| 风险指数 | 合规风险程度 | 达到100%则失败 |

### 职级晋升路径
```
Analyst（初级分析师）
    ↓
Associate（高级分析师）
    ↓
VP（副总裁）
    ↓
Director（董事）
    ↓
MD（董事总经理）
    ↓
Partner（合伙人）
```

### 可用行动

| 行动 | 消耗AP | 效果 | 解锁条件 |
|------|--------|------|----------|
| 推项目 | 1 | 业绩+50，能力+3，精力-10，风险+5% | 所有职级 |
| 拉通资源 | 1 | 高层关系+10，人脉+5，金钱-5k | VP及以上 |
| 摸鱼回血 | 1 | 精力+15，业绩-20，能力-2 | 所有职级 |
| 专业进修 | 1 | 能力+8，精力-5，金钱-10k | 所有职级 |
| 行业社交 | 1 | 人脉+15，高层关系+5，金钱-15k | Associate及以上 |
| 风险交易 | 1 | 随机收益/损失，风险+15% | VP及以上 |
| 团队建设 | 1 | 团队士气+10，业绩+10，金钱-20k | VP及以上 |

### 胜利条件
- 🎯 **职业巅峰**：晋升至合伙人(Partner)级别
- 💰 **财富自由**：累计个人资产达到1亿元
- 🏆 **行业传奇**：同时拥有声誉≥90且专业能力≥90

### 失败条件
- 😵 **身心崩溃**：精力值归零
- 🚫 **行业除名**：声誉归零
- ⚖️ **法律制裁**：风险指数达到100%且触发监管调查
- 💸 **财务破产**：负债超过500万元

```
game/

```

### 核心类说明（game.js）

#### 1. GameState（游戏状态类）
管理所有核心数值和游戏进度
- `energy`, `reputation`, `executive` 等核心数值
- `careerLevel`, `year`, `quarter` 等进度信息
- `clampStats()` - 限制数值在合法范围
- `checkGameOver()` - 检查胜利/失败条件

#### 2. UIManager（界面管理类）
处理所有界面更新
- `updateStats()` - 更新数值显示
- `updateActionButtons()` - 更新行动按钮
- `showEvent()` - 显示事件弹窗
- `showEnding()` - 显示结局

#### 3. GameController（游戏控制器）
协调整体游戏流程
- `startGame()` - 初始化游戏
- `endQuarter()` - 结束季度逻辑
- `tryTriggerEvent()` - 触发随机事件

## 如何添加新事件

事件定义在 `game.js` 的 `EVENTS` 数组中。添加新事件的步骤：

1. 在 `EVENTS` 数组中添加新对象：

```javascript
{
    id: 'your_event_id',              // 唯一标识
    title: '事件标题',
    description: '事件描述文本...',
    triggerCondition: (gameState) => {  // 触发条件
        // 返回true表示可以触发
        return gameState.risk > 50;
    },
    choices: [                        // 选项列表
        {
            text: '选项一',
            effects: {                // 数值影响
                energy: -10,
                reputation: 5,
                money: -20000
            },
            narrative: '选择后的结果描述...'
        },
        {
            text: '选项二',
            effects: {
                energy: 5,
                risk: 10
            },
            narrative: '选择后的结果描述...'
        }
    ]
}
```

2. 保存文件并刷新浏览器即可看到新事件

## 如何添加新行动

行动定义在 `game.js` 的 `ACTIONS` 数组中：

```javascript
{
    id: 'your_action_id',
    name: '行动名称',
    description: '行动描述',
    costAP: 1,              // 消耗的行动点
    minLevel: 0,            // 最低职级要求（0=Analyst, 1=Associate, ...）
    effects: {              // 数值影响
        energy: -10,
        performance: 30,
        money: -15000
    }
}
```

## 如何调整数值平衡

所有数值配置在 `game.js` 的 `CONFIG` 对象中：

```javascript
const CONFIG = {
    MAX_STAT: 100,                      // 大多数数值的上限
    MAX_PERFORMANCE: 200,               // 业绩上限
    INITIAL_STATS: {                    // 初始数值
        energy: 90,
        reputation: 70,
        // ... 其他数值
    },
    ACTION_POINTS_PER_QUARTER: 4       // 每季度行动点
};
```

## 常见问题

**Q: 为什么有些行动按钮是灰色的？**
A: 那些行动的职级要求未满足，或者当前行动点不足。

**Q: 风险交易为什么有时成功有时失败？**
A: 这是随机事件，50%概率成功，成功获利150k，失败损失80k。

**Q: 如何晋升？**
A: 每年Q4结束后会自动检查晋升条件，满足专业能力、业绩和高层关系要求即可晋升。

**Q: 业绩为什么每年都清零？**
A: 这是年度结算机制，但根据业绩会发放奖金（业绩 × 1000元）。
